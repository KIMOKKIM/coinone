당신은 Python 기반 자동매매 시스템을 설계·구현하는 시니어 개발자입니다.
목표는 Coinone API를 사용하는 BTC 현물(비레버리지) 자동매매 프로그램을 만드는 것입니다.

중요:
- 레버리지 금지
- 선물 금지
- 공매도 금지
- 현물만 사용
- 과최적화 금지
- 실제 주문 전에 반드시 백테스트와 페이퍼트레이딩 모드 지원
- 실거래와 백테스트에서 동일한 전략 로직을 재사용할 것
- 코드 전체를 모듈화하고, 유지보수 가능하게 작성할 것
- main.py 하나만 만드는 것이 아니라, 프로젝트 구조를 분리해서 작성할 것
- 모든 주요 함수에 한글 주석 추가
- .env를 사용하여 API 키 관리
- requirements.txt 생성
- 실행 방법 README.md 작성

프로젝트 목표:
최근 1개월 BTC 장세에 맞는 현실적인 비레버리지 현물 전략을 구현한다.
단순히 “30% 목표”만 추구하지 말고, 실제 시장에서 적용 가능한 진입/분할매수/분할매도/손절/리스크관리 규칙을 코드로 구현한다.

거래소 및 데이터:
- 거래소: Coinone
- 종목: BTC-KRW 또는 Coinone BTC 원화마켓 기준
- 분석 데이터와 주문 데이터를 반드시 동일 거래소(Coinone) 기준으로 사용할 것
- 캔들 데이터: 4시간봉 중심
- 보조 확인용: 일봉도 함께 사용 가능하도록 확장성 있게 설계
- 데이터 수집 실패 시 재시도 로직 구현
- API 오류, 네트워크 오류, 주문 실패, 잔고 부족 예외처리 철저히 구현

전략 설계 원칙:
1안과 2안의 장점을 결합하되, 최근 BTC 장세에 맞게 2안 중심으로 구현할 것.

========================
[매수 전략]
========================

A. 추세 필터
- 현재 가격이 4시간봉 기준 SMA50 위에 있을 때만 매수 허용
- 추가 필터로 EMA20도 계산
- EMA20 아래에서는 신규 진입 금지
- 즉, “상승 추세 + 단기 회복” 환경에서만 매수

B. 급락 후 회복 진입
- 하루 급락이 나온 직후 바로 매수하지 않는다
- 급락 다음 캔들 또는 다음 날 종가/마감 기준 회복이 확인될 때만 진입
- 급락 정의는 최근 N개 캔들 평균 변동폭 대비 큰 하락으로 파라미터화
- 예: 직전 24시간 하락폭이 임계값 이상이면 급락으로 판단

C. 20EMA 회복 진입
- 가격이 EMA20 아래에 있다가 다시 상향 돌파하고
- 최소 1개 이상의 4시간봉 또는 설정 가능한 유지 조건을 만족하면 1차 진입

D. 볼린저 밴드 돌파 진입
- 볼린저 밴드(기본 20, 2)를 계산
- 캔들 몸통 기준으로 상단 밴드를 강하게 돌파 마감하면 돌파 신호로 판단
- 단, 거래량 증가가 동반되어야 함
- 거래량은 최근 N개 평균 대비 증가 여부로 판단

E. 변동성 돌파 보조 필터
- K값 기반 변동성 돌파 전략을 보조 조건으로 사용
- 기본값 K=0.5, config에서 변경 가능
- 목표 돌파 가격 = 당일 시가 또는 기준 시가 + (전일 고가-저가) * K
- 현재가가 이 값을 상향 돌파할 때만 진입 허용

F. 주요 가격대 필터
- 최근 장세 특성을 반영해 70,000달러대 저항 개념을 KRW 기준으로 일반화하지 말고
- 절대값 하드코딩 대신 “최근 30일 고점/중간 저항/박스 상단”을 계산하는 방식으로 구현
- 예:
  - 최근 30일 최고가
  - 최근 30일 최저가
  - 최근 30일 range의 60%, 80%, 90% 구간
- 박스 상단 부근에서는 신규 추격매수 제한

========================
[분할매수 전략]
========================

- 한 번에 전량 매수 금지
- 기본 3회 분할매수 구조
- 총 투자 가능 현금 중 1회당 기본 진입 비중은 config에서 설정
- 기본값 예시:
  - 1차 30%
  - 2차 30%
  - 3차 40%

1차 매수:
- 급락 후 회복 확인 시

2차 매수:
- EMA20 회복 및 유지 확인 시

3차 매수:
- 볼린저 상단 돌파 + 거래량 증가 + 눌림목 재확인 시

추가 조건:
- 이미 포지션이 있는 상태에서만 2차/3차 매수 가능
- 평균단가와 현재 손익률을 고려하여 무리한 물타기 금지
- 하락 중 추가매수 금지
- “회복 확인 후 추가매수”만 허용

========================
[매도 전략]
========================

최근 1개월 BTC 장세 특성상, 30% 단일 익절보다 구간 분할매도가 더 현실적이므로 둘 다 옵션으로 지원할 것.

A. 기본 분할익절
- 1차 익절: +5% ~ +7%
- 2차 익절: +8% ~ +12%
- 3차 익절: +12% ~ +15%
- 각 구간별 매도 비중 config로 설정 가능
- 기본 예시:
  - 1차 30%
  - 2차 30%
  - 3차 40%

B. 30% 목표 모드
- 옵션으로 “aggressive_mode” 또는 “target_30_mode” 지원
- 이 모드에서는 +30% 도달 시 전량 매도
- 단, 기본 모드는 아님
- 기본값은 OFF

C. 트레일링 스탑
- 가격 상승 후 볼린저 밴드 중간선(20선) 이탈 시 잔여 물량 청산
- 또는 최고가 대비 일정 % 하락 시 추적 손절 가능
- 두 방식 모두 선택 가능하도록 설정

D. 저항대 기반 매도
- 최근 30일 최고가 근처 또는 박스 상단 접근 시 부분익절
- 최근 고점 재도전 실패 시 추가 매도

E. 급등 후 약세 전환 매도
- 1일 또는 설정 기간 기준 급등 후 다음 캔들이 약세 전환(음봉, 거래량 감소, 상단 꼬리 확대 등)일 경우 부분익절

========================
[손절 및 리스크 관리]
========================

A. 손절
- 진입가 대비 -3% ~ -5% 손절 범위 지원
- 전략별 손절 방식 분리:
  1) EMA20 회복 매수 후 EMA20 재이탈 시 손절 또는 축소
  2) 돌파매수 후 돌파 전 가격대로 복귀 시 손절
  3) 고정 손절 % 도달 시 즉시 손절

B. 자금 관리
- 한 번의 거래에 전체 자금 올인 금지
- 초기 설정 기본값:
  - 총 자산의 최대 20%~35%만 한 사이클에 사용
- 전체 잔고 대비 최대 포지션 비율 제한 기능 구현

C. 이벤트 리스크 회피
- 거시 이벤트 전날 신규 진입 제한 기능을 옵션으로 설계
- 실제 경제지표 API 연동이 없더라도, 추후 추가 가능하도록 인터페이스 분리
- 현재는 수동 입력 방식 또는 일정 차단 설정 가능하게 구현

D. 재진입 제한
- 손절 직후 연속 재진입 금지
- 쿨다운 시간 설정 가능
- 예: 손절 후 12시간 또는 24시간 동안 신규 진입 금지

구현 요구사항:
1. coinone_client.py
- 캔들 조회
- 현재가 조회
- 잔고 조회
- 시장가/지정가 매수/매도
- 주문 가능 여부 확인
- API 예외처리
- 주문 권한 체크 함수 추가

2. indicators.py
- SMA
- EMA
- Bollinger Bands
- ATR 또는 변동성 측정
- 거래량 평균
- 최근 30일 고점/저점/박스 구간 계산

3. signal_engine.py
- 매수/매도 신호 생성
- 전략 조건을 개별 함수로 분리
- 각 조건의 충족 여부를 dict 형태로 상세 반환
- 예:
  {
    "trend_ok": True,
    "ema_reclaim_ok": False,
    "bb_breakout_ok": True,
    "volume_ok": True,
    "volatility_breakout_ok": False,
    "final_buy_signal": True
  }

4. risk_manager.py
- 포지션 크기 계산
- 최대 손실 제한
- 손절/익절/트레일링스탑 처리
- 쿨다운 처리

5. portfolio.py
- 평균매수가
- 보유수량
- 평가손익
- 분할매수/분할매도 상태 추적
- 이미 1차/2차/3차 매수 또는 매도를 했는지 상태 저장

6. trader.py
- 실거래 실행
- 신호 기반 주문
- 중복 주문 방지
- 슬리피지와 최소 주문금액 고려
- 로그 저장

7. backtester.py
- 동일 전략으로 과거 데이터 백테스트
- 총 수익률
- MDD
- 승률
- 평균 수익/손실
- 거래 횟수
- 샤프 비율(가능하면)
- 결과 CSV 저장

8. notifier.py
- 텔레그램 또는 콘솔 알림 구조
- 최소한 콘솔에 자세한 로그 출력
- 예:
  - 매수신호 발생
  - 1차 매수 실행
  - 손절 실행
  - 익절 실행
  - API 오류

9. scheduler.py
- 4시간봉 마감 시점에 맞춰 전략 실행
- 테스트 모드에서는 즉시 1회 실행 가능

========================
[설정값(config)]
========================

다음 설정값은 config.py 또는 .env에서 조정 가능하게 구현:

- SYMBOL = "BTC"
- MARKET = "KRW"
- TIMEFRAME = "4h"
- SMA_PERIOD = 50
- EMA_PERIOD = 20
- BB_PERIOD = 20
- BB_STD = 2
- VOL_BREAKOUT_K = 0.5
- STOP_LOSS_PCT = 0.04
- TAKE_PROFIT_LEVELS = [0.05, 0.10, 0.15]
- TAKE_PROFIT_RATIOS = [0.3, 0.3, 0.4]
- MAX_CAPITAL_ALLOCATION = 0.35
- BUY_SPLIT_RATIOS = [0.3, 0.3, 0.4]
- COOLDOWN_HOURS = 12
- ENABLE_TARGET_30_MODE = False
- TARGET_30_PCT = 0.30
- PAPER_TRADING = True

[상태 저장]
========================

- 현재 포지션 상태를 파일(JSON/SQLite 등)로 저장

[안전장치]
========================

- PAPER_TRADING=True일 때는 실제 주문 금지
- 실거래 전환 시 경고 로그 출력

- 로그 파일과 콘솔에 모두 기록
